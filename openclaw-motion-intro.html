capability matrix

保留远端 Web 体验，把桌面端可靠性补齐。

OpenClaw Manager 是薄壳，不在本机托管 Gateway，也不重写前端。它负责窗口、会话、恢复、诊断和本地状态，让远程控制台具备桌面应用该有的稳定边界。

webview.profile()

环境隔离

每个 Gateway 环境使用独立 WebView2 profile，切换远端实例时不混用 cookie、缓存和登录态。

heartbeat.observe()

连接感知

状态条显示加载、鉴权、工作态和延迟，让远端 WebSocket 与代理抖动更可见。

recover.softFirst()

轻量恢复

优先页面内 reconnect 与 lightweight sync，只有软恢复失败时才刷新或重建 WebView。

diagnostics.run()

诊断闭环

WebView2 runtime、HTTP 可达性、Control UI session 状态和日志入口集中到一个地方。

recovery state machine

恢复状态机先判断，再决定是否打断用户。

面向 Cloudflare Tunnel、反向代理和远端 Gateway 的现实网络，恢复策略是一条分级链路，而不是遇到抖动就简单刷新页面。

观察

Control UI bridge、heartbeat 和 navigation event 持续提供状态信号。

判定

区分登录失效、origin 拒绝、Gateway reconnect 和普通网络失败。

软同步

session 仍可用时优先 fetch recent messages 或 lightweight sync。

重连

向页面发送 reconnect intent，并等待 hosted UI 自己恢复。

刷新

只有软恢复失败时才进入 reload 或 WebView 重建路径。

local data boundary

本地数据边界清楚，远端计算仍留在 Gateway。

运行时数据留在用户本机的 LocalAppData；远端计算、模型和 Gateway runtime 仍由远端 OpenClaw 部署负责。桌面壳只保存使用体验所需的状态和诊断线索。

settings 保存环境列表、主题、语言、窗口状态和恢复策略。 local json
logs 按日期写入结构化日志，便于回看连接、恢复和诊断事件。 daily files
WebView2Data 每个环境一份浏览器 profile，管理 cookie、缓存和本地会话。 per env

native entry point

远端 OpenClaw 的 Windows 原生控制台入口。

打开就是控制台，切换就是新环境，异常时有诊断，断线时有恢复。OpenClaw Manager 提供的是一个安静但可观察的桌面入口。
