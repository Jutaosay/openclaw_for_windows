OpenClaw Manager

一个轻量的 Windows 原生管理入口，把远端 OpenClaw Control UI 承载在稳定、可恢复、可诊断的桌面体验里。适合 VPS、Cloudflare Tunnel、反向代理后的远程 Gateway。

承载的能力

保留 Web 体验，补上桌面端该有的可靠能力。

OpenClaw Manager 不重写 Control UI，也不在本地托管 Gateway。它负责窗口、会话、恢复、诊断和本地状态，让远端控制台用起来像一个真正的 Windows 应用。

webview.profile()

环境隔离

每个 Gateway 环境使用独立 WebView2 profile，切换远端实例时不混用 cookie、缓存和登录态。

heartbeat.observe()

连接感知

状态条显示加载、连接、鉴权、工作态和延迟，让远端 WebSocket 与代理抖动更可见。

recover.softFirst()

恢复优先级

先尝试页面内 reconnect 和轻量同步，必要时才 reload，尽量减少正在输入时的打扰。

diagnostics.run()

诊断闭环

WebView2 runtime、HTTP 可达性、Control UI session 状态和日志入口集中到一个地方。

恢复链路

当远端链路短暂抖动，应用先稳住，再决定要不要重载。

面向 Cloudflare Tunnel、反向代理和远端 Gateway 的现实网络，恢复策略更像一个分级流程，而不是简单刷新页面。

观察

Control UI bridge、heartbeat、navigation event 持续提供状态信号。

判定

区分登录失效、origin 拒绝、Gateway reconnect、普通网络失败。

软同步

session 还活着时优先 fetch recent messages 或 lightweight sync。

重连

向页面发送 reconnect intent，并等待 hosted UI 自己恢复。

刷新

只有软恢复失败时才进入 reload 或 WebView 重建路径。

本地占用

轻量，边界清楚。

运行时数据留在用户本机的 LocalAppData；远端计算、模型和 Gateway runtime 仍由远端 OpenClaw 部署负责。

settings 保存环境列表、主题、语言、窗口状态和恢复策略。
logs 按日期写入结构化日志，便于回看连接与恢复事件。
WebView2 每个环境一份浏览器 profile，管理 cookie、缓存和本地会话。
原生入口

远端 OpenClaw 的 Windows 原生入口。

打开就是控制台，切换就是新环境，异常时有诊断，断线时有恢复。OpenClaw Manager 提供的是一个安静但可靠的入口。
